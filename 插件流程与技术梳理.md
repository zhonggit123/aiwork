# 题库录入 AI 插件 — 流程与技术梳理

> 用于合成一张说明图：解决问题流程 + 所用技术栈。

---

## 一、解决的问题

- **场景**：从 Word/PDF 题目文档 → 自动识别、解析为结构化题目数据 → 填入「驰声录题」等录题系统页面。
- **痛点**：手工逐题复制粘贴、对照填表效率低；题目格式多样（题干、选项、答案、解析、听力原文、填空、图片选项等），需要智能解析与页面结构适配。

---

## 二、整体流程（三步）

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  步骤 1：分析页面                                                             │
│  用户在录题页点击插件 → 插件检测当前页是否为录题页 → 遍历表单，识别每题的      │
│  输入框/上传框（题干、选项、答案、解析、听力原文、音频、图片等）→ 得到「选择器  │
│  映射」和「每题结构」（小题数、选项是文字还是图片等）→ 结果存到 storage       │
└─────────────────────────────────────────────────────────────────────────────┘
                                        ↓
┌─────────────────────────────────────────────────────────────────────────────┐
│  步骤 2：选择文件 → AI 解析                                                    │
│  用户在插件里选择 Word(.docx) 文件（可多选）→ 插件把文件 + 页面结构信息        │
│  发给本地后端 → 后端读 Word 全文、按块切分、提取图片 → 调用大模型 API 解析     │
│  为结构化 JSON（题干/选项/答案/解析/听力/填空等）→ 若有图片选项，用占位符      │
│  <<IMG>> 后由后端把 Word 图片写入临时目录并生成可访问 URL → 返回题目列表       │
└─────────────────────────────────────────────────────────────────────────────┘
                                        ↓
┌─────────────────────────────────────────────────────────────────────────────┐
│  步骤 3：解析并填入                                                            │
│  用户点击「立即填入当前页面」→ 插件把题目 JSON + 选择器发给录题页的 Content    │
│  Script → Content Script 按题逐空填充：文本框填文字、文件框填音频/图片 URL     │
│  （本地图片通过后端 /api/images 提供）→ 必要时调用后端 TTS 合成听力音频 →      │
│  填完一题可自动点「下一题」继续，直到本页题目填完                              │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 三、流程细化（角色与数据流）

| 阶段 | 发生位置 | 动作 | 数据流 |
|------|----------|------|--------|
| **1. 检测录题页** | Popup + Content Script | 根据 URL 判断是否为录题页（如 91tszx / chisheng）；若是，发 `DETECT_AND_WALK` 到当前 tab | 当前 tab 的 DOM → 遍历表单 → 得到 selectors、pageSlots、pageQuestionTotal → 存 `chrome.storage.sync` |
| **2. 上传与解析** | Popup + Background + 本地后端 | 用户选 Word → Background 调 `POST /api/parse-multiple`，带 `field_structure`、`page_structure`、`expected_total` 等 | FormData(files + 页面结构) → 后端读 Word、提图片、调 LLM → 返回 `{ questions, session_id?, debug_info? }` |
| **3. 填入表单** | Popup/Background + Content Script | 用户点「填入」→ 发 `FILL_FORM` 到录题页 tab，带 questions、selectors、defaultAudioUrl 等 | questions 按 pageSlots 规范为大题列表 → Content Script 根据 selectors 找元素，填 value/上传文件/TTS 合成 → 逐题提交 |

**补充能力：**

- **高级 JSON**：用户可粘贴已有 JSON（如接口返回的 `topic`），直接「仅填入当前页面」，不经过 Word 解析。
- **TTS**：题干/听力原文等可走后端火山引擎 TTS 合成 mp3，再填入音频上传框；支持 W:/M:/Q:/A: 对话标记分男女声。
- **图片**：Word 内图片被提取后存到后端临时目录，通过 `http://localhost:8766/api/images/{session_id}/{filename}` 提供；填充时遇到该类 URL 可转成文件上传到录题页。

## 四、技术栈总览

| 层次 | 技术 | 说明 |
|------|------|------|
| **浏览器端** | Chrome Extension (Manifest V3) | Popup（popup.html/js）、Background Service Worker（background.js）、Content Script（content.js）、Options 页 |
| **前端能力** | JSZip、chrome.storage、chrome.tabs、sidePanel | 解压 docx、持久化选择器与状态、与录题页通信、侧边栏常驻 |
| **录题页交互** | DOM 遍历、fetch/XHR 注入、表单填充 | 注入脚本监听接口返回的 `topic`；根据选择器填 input/textarea/上传框；TTS 未合成时调后端 `/api/tts` |
| **本地后端** | FastAPI + Uvicorn (端口 8766) | 提供 /api/parse、/api/parse-multiple、/api/tts、/api/images、/api/config、/api/submit 等 |
| **Word 解析** | python-docx、Pillow | 读 docx 全文与段落、提取图片、表格转图片；TIFF/CMYK 等转 JPEG 以兼容浏览器 |
| **AI 解析** | OpenAI 兼容 API（默认火山引擎豆包）、json-repair | 按块/按文件类型（试题/答案/合并）发 prompt，强制 JSON 输出；题干/选项/答案/解析/听力/填空等结构化；★ 占位符防误删 |
| **TTS** | 火山引擎 TTS v3（seed-tts-1.0 / seed-tts-2.0） | 对话模式 W:/M:/Q:/A: 分男女声，支持语速/音量；返回 base64 音频给前端或 Content Script |
| **提交（可选）** | requests、Playwright | 若用接口提交：抓包得到 POST URL/headers/body，直接 requests；若用浏览器填表：Playwright 打开录题页按选择器填表 |

## 五、关键文件与接口

| 文件/接口 | 作用 |
|-----------|------|
| `extension/manifest.json` | 扩展权限、content_scripts、host_permissions（含 localhost:8765/8766） |
| `extension/popup.js` | 检测录题页、发起 DETECT_AND_WALK、上传文件、调 background 解析、发起填入、TTS 设置与试听 |
| `extension/background.js` | 调用后端 `/api/parse-multiple`、处理填入请求（FILL_FORM）、角标、取消解析 |
| `extension/content.js` | DETECT_FORM / DETECT_AND_WALK（遍历表单）、FILL_FORM（按 selectors 填表）、GET_PAGE_STRUCTURE、GET_FORM_AS_JSON、topic 接口捕获、TTS/图片上传逻辑 |
| `app.py` | FastAPI：parse/parse-multiple、tts、images、config、submit；Word 图片会话管理、选项图注入 |
| `llm_extract.py` | 文件类型快速分类、单文件/题答合并/多文件解析、调用 OpenAI 兼容 API、★ 与填空/对话格式后处理 |
| `word_reader.py` | 读 docx 全文、按块切分、提取图片与表格为图片 |
| `config.yaml` | llm（base_url、api_key、model）、可选 submit（api/playwright） |

## 六、数据格式约定（与录题页的桥梁）

- **插件 → 录题页**：题目 JSON 含 `question`、`options`、`answer`、`explanation`、`listening_script`、`blanks`（小题）、`audio_url`/`image_url` 等；选项可为文字或图片 URL（含 `http://localhost:8766/api/images/...`）。
- **录题页 → 插件**：通过「获取当前题 JSON」可把接口返回的 `topic`（如 get?topicType=...&topicID=...）转成上述统一 JSON（见 `API_TOPIC_RULES.md`），便于粘贴到高级 JSON 或对比。
