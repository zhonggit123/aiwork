# 题库后端 - 打包成 exe 与分发

把 Python 后端打成单个 exe，对方**无需安装 Python**，双击即可启动服务；再配合浏览器插件即可使用。

## 一、如何打包

### 方式 A：没有 Windows 电脑（用 GitHub 云端打包）

1. 把本项目推到 **GitHub**（或 Gitee 等支持 Actions 的仓库）。
2. 在仓库网页打开 **Actions**，左侧选 **「打包 Windows exe」**。
3. 点击 **「Run workflow」** → **「Run workflow」**，等几分钟。
4. 跑完后点进这次运行，在页面底部 **Artifacts** 里下载 **「QuestionBank-Backend-Windows」**（纯英文名避免下载报错），解压即得 `题库后端.exe`。

这样无需本机装 Windows，由 GitHub 的 Windows 虚拟机帮你打好包。

### 方式 B：本机是 Windows

1. 安装 PyInstaller：`pip install pyinstaller`
2. 在项目目录执行：`pyinstaller 题库后端.spec`
3. 生成文件在：`dist\题库后端.exe`

### 方式 C：本机是 Mac / Linux

在项目目录执行 `pyinstaller 题库后端.spec` 会生成**当前系统**的可执行文件（Mac 是 `dist/题库后端`，无 .exe 后缀）。要得到 **Windows 的 .exe** 请用方式 A。

### 生成结果说明

- **单文件 exe**：`题库后端.exe`（或 Mac 上为 `题库后端`）。
- 首次运行会在 exe 同目录生成 `config.yaml`（从内嵌的 config.example.yaml 复制），用户编辑该文件填写 API 等配置即可。

## 二、发给别人：需要给哪些东西

1. **`dist/题库后端.exe`**（一个文件即可，静态资源和配置模板已内嵌）
2. **插件**：把项目里的 `extension` 文件夹打成 zip 发给对方，让对方按 `extension/安装说明.md` 用 Chrome「加载已解压的扩展程序」安装。

## 三、对方电脑上的使用步骤

1. **配置 API（首次使用）**
   - 双击运行 `题库后端.exe`。
   - 若同目录下没有 `config.yaml`，程序会自动生成一份；**关闭程序**，用记事本等编辑同目录下的 `config.yaml`，填写大模型 API（如豆包）的 `base_url`、`api_key`、`model` 等（可参考文件内注释）。
   - 再次双击 `题库后端.exe` 启动服务，保持窗口不关闭。

2. **安装浏览器插件**
   - 解压你发的 `extension.zip`，在 Chrome 打开 `chrome://extensions/`，开启「开发者模式」，点击「加载已解压的扩展程序」，选择解压后的 `extension` 文件夹。
   - 将「题库自动填表」固定到工具栏。

3. **使用**
   - 打开你们的录题页面（如 cp.91tszx.com），点击插件图标，上传 Word 即可；插件会请求本机 `http://127.0.0.1:8766`，即本机正在运行的 `题库后端.exe`。

## 四、注意事项

- 对方电脑**不需要安装 Python**，只需保留 exe 与同目录下的 `config.yaml`。
- 若杀毒软件提示「未知发行商」，属正常（未签名），可添加信任或暂时关闭杀毒再运行。
- 端口 **8766** 需未被占用；若被占用，需改 exe 启动端口（目前写死在 `run_server.py` 的 8766，若要可配置可再改一版）。
